# 四種筆記類型概覽

| 類型 | 符號 | 用途 | 特性 | 存放位置 |
|-----|------|------|------|---------|
| 來源筆記 | 📥 | 記錄外部來源的摘要 | 暫時性、需標註來源 | `01_Inbox/` |
| 名詞筆記 | 📖 | 定義和解釋單一概念 | 原子性、可重組、用自己的話 | `02_Notes/` |
| 關係筆記 | 🔗 | 探討概念間的關聯 | 連結多個概念、產生新洞察 | `02_Notes/` |
| 主題筆記 | 🗺 | 特定主題的導航頁面 | 組織連結、展示結構 | `03_Maps/` |

# 來源筆記 📥

## 定義

記錄**外部來源**（書籍、文章、影片、課程、演講、LLM 問答等）的內容摘要和個人想法。

## 使用情境

- 閱讀書籍時
- 看完文章或影片後
- 參加課程或講座時
- 與 LLM 問答完之後

## 範例

- `[書]《原子習慣》- 查核與執行意圖`
- `[影片] 李永樂老師講晶片 - 什麼是光刻機`
- `[文章] 為什麼我們需要卡片盒筆記法`

## 核心原則

1. **標註來源資訊** : 作者、標題、日期等
2. **忠實捕捉** : 原則是**不失真**地記錄從「外部」(書籍、課程、文章) 獲得的資訊，重點在於「這是別人說的」
3. **快速反應** : 捕捉的同時，寫下「當下」的初步想法、疑問或心得
4. **加入待辦事項** : 提醒自己後續處理事項，例如新增某個名詞筆記
5. **禁忌** : 不要美化、整理，價值在於原始與完整

# 名詞筆記 📖

## 定義

用於定義和解釋**單一概念、術語或理論**的筆記。

## 使用情境

- 遇到具體的、單一的、需要被定義的概念
- 專業術語、可獨立解釋的名詞
- 需要用自己的話重新解釋的概念

## 範例

- `捷思法 (Heuristics)`
- `滾動波動率 (Rolling Volatility)`
- `卡片盒筆記法 (Zettelkasten)`

## 核心原則

1. **原子性** : 一個筆記只聚焦一個核心概念
2. **可複用** : 名詞筆記的目的是被重複連結
3. **用自己的話重新詮釋** : 以自己的方式理解名詞，而非純粹複製定義
4. **使用範例** : 加入例子幫助理解、記憶，愈多例子愈好
5. **禁忌** : 不要放入個人觀點或論述，例如 : `[名詞]Promise` 筆記只該解釋什麼是 Promise，而不該討論「Promise 是不是 JS 最好的異步方案」(那是 `[主題筆記]` 的工作)

# 關係筆記 🔗

## 定義

探討**概念之間的關係、對比、衝突或整合**，並產生新的洞察。

## 使用情境

- 發現兩個概念有相似或對比之處
- 想要整合多個想法
- 產生我自己的想法時

## 範例

- `為什麼「滾動波動率」是判斷市場恐慌的有效指標` (這張筆記會連結到 `[名詞]滾動波動率` 和 `[來源]某某財經文章`)
- `如何利用「捷思法」的偏誤來設計行銷文案` (這張筆記會連結到 `[名詞]捷思法` 和 `[名詞]定錨效應`)
- `遊戲開發中「敏捷開發」的必要性`

## 核心原則

1. **發展論點** : 唯一真正屬於自己想法的筆記，它的標題通常是一個「陳述句」或「你的觀點」，而不只是一個名詞
2. **串聯證明** : 連結至少 2 個以上的概念，說明為什麼這些概念有關聯
3. **禁忌** : _不要_ 建立一張「無法連結」其他筆記的主題筆記。如果一張主題筆記裡沒有連結到任何 `[名詞]` 或 `[來源]`，那它可能只是一個空泛的想法，或是應該被拆解成 `[名詞筆記]`

# 主題筆記 🗺

## 定義

作為特定主題的**導航頁面和概覽**（Map of Content, MOC）。

## 使用情境

- 某個主題下累積了 10+ 則相關筆記
- 需要整理某個領域的知識結構
- 進行專案或學習計畫時

## 範例

- `MOC - 投資學`
	- 連結 `[主題] 如何評估公司價值`
	- 連結 `[名詞] 護城河`
	- 連結 `[名詞] 滾動波動率`
- `MOC - 遊戲開發`
	- 連結 `[主題] 遊戲開發中的敏捷...`
	- 連結 `[名詞] Unity 引擎`
	- 連結 `[來源] GDC 2025 演講筆記`

## 核心原則

1. **組織結構** : 唯一目的是組織其他三種筆記，展示該主題下的知識結構
2. **持續成長** : 可隨著知識增長持續更新
3. **禁忌**，不在這張筆記裡寫下任何具體內容，他只對筆記做連結

# 實戰流程

## 基本流程

```
📥 來源筆記 (01_Inbox)
    ↓ 提煉
📖 名詞筆記 (02_Notes)
    ↓ 連結
🔗 關係筆記 (02_Notes)
    ↓ 組織
🗺 主題筆記 (03_Maps)
```

## 詳細步驟

1. 輸入階段：從來源開始
	- 閱讀書籍、文章時，在 `01_Inbox/` 創建**來源筆記**
	- 記錄摘要、想法和反思
	- 在筆記中加入「待提煉」清單
2. 提煉階段：建立概念
	- 定期回顧 `01_Inbox/` 的來源筆記
	- 將重要概念提煉成**名詞筆記**，放入 `02_Notes/`
	- 用自己的話重新詮釋，加入例子
	- 確保每個筆記只包含一個核心概念
3. 連結階段：發現關係
	- 在寫筆記時主動尋找連結
	- 當發現兩個概念有關聯時，在 `02_Notes/` 創建**關係筆記**
	- 重點不是「連結」本身，而是「為什麼連結」
	- 產生原創洞察
4. 組織階段：建立地圖
	- 當某個主題累積 10+ 則筆記時，在 `03_Maps/` 創建**主題筆記**
	- 用於導航和概覽，不包含深度內容
	- 隨著知識增長持續更新

## 決策流程

創建新筆記時的判斷流程

```
┌─────────────────────────────────────┐
│  我要創建一則新筆記                    │
└─────────────┬───────────────────────┘
              │
              ▼
     ┌────────────────────┐
     │ 這是直接從書籍/文章  │       是
     │ /影片摘錄的嗎？      │─────────► 📥 來源筆記
     └────────┬───────────┘              放入 01_Inbox/
              │ 否
              ▼
     ┌────────────────────┐
     │ 這是單一概念的       │       是
     │ 定義或解釋嗎？       │─────────► 📖 名詞筆記
     └────────┬───────────┘              放入 02_Notes/
              │ 否
              ▼
     ┌────────────────────┐
     │ 這是連結多個概念     │       是
     │ 並產生新想法嗎？     │─────────► 🔗 關係筆記
     └────────┬───────────┘              放入 02_Notes/
              │ 否
              ▼
     ┌────────────────────┐
     │ 這是用來組織導航     │       是
     │ 一群相關筆記嗎？     │─────────► 🗺 主題筆記
     └────────────────────┘              放入 03_Maps/
```

# 其他建議

## 時間分配

- **每日**：創建 1-2 則來源筆記或名詞筆記（10-15 分鐘）
- **每週**：
	- 回顧本週筆記，創建 2-3 則關係筆記（30 分鐘）
	- 清理 `01_Inbox/`，提煉重要內容（15 分鐘）
- **每月**：
	- 檢視整體結構，更新或創建主題筆記（1 小時）
	- 回顧和更新現有筆記（30 分鐘）

## 補充 Tags

可以加入額外的分類標籤：

```markdown
---
tags:
  - 📖
  - 軟體設計
  - 設計模式
  - 重要
---
```

## 使用模板

在 Obsidian 中設定模板資料夾，為每種筆記類型創建模板：

**模板資料夾結構**：
```
_templates/
├── template_名詞筆記.md
├── template_來源筆記.md
├── template_關係筆記.md
└── template_主題筆記.md
```

## 建立總索引

當 MOC 筆記累積到一定量時，可以建立 `總索引 MOC.md`

```markdown
---
tags:
  - 🗺
  - 索引
---
# 總索引 MOC

> 整個知識系統的入口點

## 主要領域
- [[個人知識管理 MOC]] (03_Maps/)
- [[程式設計 MOC]] (03_Maps/)
- [[心理學 MOC]] (03_Maps/)

## 近期關注
- [[學習 Rust 程式語言 MOC]] - 進行中
- [[心流狀態 vs 正念冥想 - 相似與差異]] - 新洞察

- [ ] 提煉《深度工作》的來源筆記
- [ ] 整理上週的會議筆記
```

## 利用 Dataview 動態查詢

```dataview
TABLE
  file.ctime as "創建日期",
  length(file.outlinks) as "連結數"
FROM "02_Notes"
WHERE contains(tags, "📖")
SORT file.mtime DESC
LIMIT 10
```

這個查詢會列出最近更新的 10 則名詞筆記及其連結數。
